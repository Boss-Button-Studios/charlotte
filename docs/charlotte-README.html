What she does

Charlotte takes a URL and a plain-language goal and navigates to find it. She evaluates each page, decides which links are worth following, and stops when she's done — or when she runs out of budget. She does not index. She does not scrape blindly.

Traditional scrapers are brittle — they break when a site rearranges its pages. Charlotte navigates by meaning. A page that renames its button from "Download Directory" to "Get the Current PDF" is still navigable.

Goal-directed

Natural language goal. One model call per page. Stops when found.

BYOM

Any instruct-tuned model. Cloud or self-hosted. Groq and Ollama ship by default.

Hard budgets

Max pages, max depth, four independent timeouts. Safe to call in automated pipelines.

Injection-resistant

Three-layer sanitization pipeline. URL provenance check. Trust model throughout.

Single or multi-result

max_results=None collects all matching links within budget.

Honest

Identifies itself in the User-Agent. Respects robots.txt by default.

Install

pip install charlotte-crawler

Playwright is optional — only required when render_js=True:

pip install charlotte-crawler[playwright]
playwright install chromium

Quick start

Find a single link

from charlotte import find_link

result = find_link(
    start_url="https://www.example.gov/services",
    goal="Find the current annual services directory PDF",
    navigation_hint="Usually linked from the Publications section"
)

if result.found:
    print(result.urls[0])  # → https://example.gov/.../directory-2026.pdf

Collect multiple links

result = find_link(
    start_url="https://www.example.gov/regions",
    goal="Find all regional directory PDFs",
    navigation_hint="Directories are in a dropdown, one per region"
    # max_results=None by default — collect all
)
# result.urls → [north.pdf, south.pdf, east.pdf, west.pdf]

Full navigation with metadata

from charlotte import crawl

result = crawl(
    start_url="https://www.lincolnhigh.edu",
    goal="Find the academic calendar page",
    navigation_hint="Usually under Parents or Academics",
    max_pages=10
)
# result.result_urls, result.visit_log, result.pages_visited ...

Self-hosted (Ollama)

from charlotte import crawl
from charlotte.adapters import LocalAdapter

result = crawl(
    start_url=url,
    goal=goal,
    model=LocalAdapter()  # points to localhost:11434 by default
)

Models

Charlotte makes one model call per page visited. Speed and cost per call matter. Any instruct-tuned model that produces reliable structured JSON output works.

ModelViaNotes
Llama 3 8B InstructGroqDefault for cloud. Fast, cheap.
Llama 3 8B InstructOllamaDefault for self-hosted via LocalAdapter.
Mistral 7B InstructGroq / Together.aiStrong alternative. Good structured output.
Llama 3 70B InstructGroq / Together.aiBetter on ambiguous navigation. Higher cost.
Claude HaikuAnthropicFast, strong instruction following.
Phi-3 MiniOllamaLighter local option. Verify output reliability.

The LocalAdapter is a fully supported production path — not a development shortcut. Self-hosted inference is the right choice for any deployment where you control the model host and want zero per-query API costs.

What Charlotte is not

A bulk crawler or site indexer
A search engine replacement
An authenticated session handler
A captcha solver or bot-detection workaround
A caching or persistence layer
An autonomous web agent that changes its own goals mid-run

Charlotte is purposefully narrow. The spec is here.

Contributing

Open an issue. Read SPEC.md before proposing changes — most design decisions are deliberate and documented there. The test matrix in Section 19 of the spec is the baseline; new features should come with new rows.

Writing an adapter for a new model provider? The contract is in Section 6.3. Adapter output validation is Charlotte's responsibility, not yours — you only need to produce the five required fields.
